DEVELOPER EXPERIENCE

Code-first. Infra later.

Define your entire infrastructure in code, get sane defaults, and override anything you need.

Deploy with a single config

One nexcore.yaml file defines services, resources, scaling rules, and env variables.

# nexcore.yaml
service: api-gateway
runtime: node20
region: [eu-west, us-east]
scale:
  min: 2
  max: 50
  metric: rps
env:
  DATABASE_URL: secret://prod/db

# Deploy in 180ms →
$ nexcore deploy --env prod
